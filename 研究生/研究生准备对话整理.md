## 8 月第 1 周：LLM 任务规划

你要做一个简单模块：

输入自然语言：

```text
把红色方块放到盒子里
```

输出结构化动作序列：

```json
[
  {"skill": "detect", "object": "red_block"},
  {"skill": "pick", "object": "red_block"},
  {"skill": "place", "target": "box"}
]
```

注意：  
你不一定非要一开始调用在线大模型。你可以先用规则写一个“假 LLM 输出”，保证系统流程跑通。

比如：

```python
def parse_instruction(text):
    if "红色" in text and "盒子" in text:
        return [
            {"skill": "detect", "object": "red_block"},
            {"skill": "pick", "object": "red_block"},
            {"skill": "place", "target": "box"}
        ]
```

先跑通流程，再接真正的 LLM API。

## 8 月第 2 周：加入状态机 / 安全检查

这是你的控制工程特色。

你设计一个有限状态机：

```text
Idle
Detected
MovingToPick
Grasping
Holding
MovingToPlace
Placed
Failed
```

动作必须合法转移：

```text
Idle → Detected → MovingToPick → Grasping → Holding → MovingToPlace → Placed
```

非法动作直接拒绝：

```text
没有抓住物体，不能 place
没有检测到物体，不能 pick
夹爪已经抓着东西，不能再 pick
目标不存在，不能执行
目标位置超出范围，不能执行
```

这就是我前面说的“控制增强”。

## 第 2 轮

### 用户提问

组内用的是ros但是我说也是，ros早就停止更新了，还用ros，我现在就是焦虑的你说我研究生三年用的ros出去外边都是要ros2的咋办

### 助手回答

你的焦虑有道理，但结论不是“组里用 ROS1 就完了”。

更准确地说：

> **组里用 ROS1，你就用 ROS1 把实验和论文做出来；但你自己必须同步学 ROS2，把能力迁移过去。**

这不是二选一，而是**实验室现实 + 个人竞争力**两条线并行。

## 先确认事实：ROS1 确实已经进入历史阶段

ROS Noetic 是最后一个 ROS1 发行版，官方已经说明它在 **2025 年 5 月**到达生命周期终点。ROS 官方也提到，过去几年重点已经转向 ROS2，并且 2024 年 9 月服务器下载数据里 ROS2 下载量已经占 ROS 下载量接近 80%。([ros.org](https://www.ros.org/blog/noetic-eol/?utm_source=chatgpt.com))

所以你担心“以后外面都用 ROS2”，这个方向判断是对的。

但是，另一个事实也很重要：

> **很多高校实验室、旧机械臂平台、老项目、老驱动仍然在用 ROS1。**

尤其是机械臂、移动机器人、仿真平台，很多现成代码、驱动包、师兄师姐遗留工程，都是 ROS1。导师和实验室不可能因为 ROS1 停更，就立刻把所有硬件驱动、代码、论文实验全部迁到 ROS2。

所以你现在不能说：

> “ROS1 过时了，我不学。”

你应该说：

> “ROS1 是我做组内实验的工具，ROS2 是我未来就业和长期发展的工具。”

---

## ROS1 学了会不会浪费？

不会。

ROS1 和 ROS2 的底层实现不一样，但很多核心思想是一样的：

| 能力 | ROS1 | ROS2 | 是否可迁移 |
|---|---|---|---|
| 节点 Node | 有 | 有 | 可迁移 |
| 话题 Topic | 有 | 有 | 可迁移 |
| 服务 Service | 有 | 有 | 可迁移 |
| Action | 有 | 有 | 可迁移 |
| 消息类型 msg/srv/action | 有 | 有 | 可迁移 |
| TF 坐标变换 | tf/tf2 | tf2 | 可迁移 |
| URDF 机器人建模 | 有 | 有 | 可迁移 |
| RViz 可视化 | RViz | RViz2 | 可迁移 |
| MoveIt 机械臂规划 | MoveIt1 | MoveIt2 | 可迁移 |
| 控制框架 | ros_control | ros2_control | 思想可迁移 |

真正不同的是：

```text
ROS1：roscore、catkin、rospy/roscpp、中心化 master
ROS2：DDS、colcon、rclpy/rclcpp、去中心化通信、QoS、生命周期节点
```

也就是说，你学 ROS1 不是在学一个废工具，而是在学机器人系统的基本组织方式。

## 最适合你的学习策略：先 ROS1，后 ROS2，对照学

因为组里用 ROS，你入组后必须能看懂师兄师姐代码，所以你不能绕开 ROS1。

但是你学 ROS1 的时候，每学一个概念，都要同步知道 ROS2 怎么写。

例如：

| 学 ROS1 时 | 立刻对应 ROS2 |
|---|---|
| catkin_make | colcon build |
| source devel/setup.bash | source install/setup.bash |
| rosrun | ros2 run |
| roslaunch | ros2 launch |
| rostopic list | ros2 topic list |
| rostopic echo | ros2 topic echo |
| rosservice list | ros2 service list |
| rospy.Publisher | rclpy Node.create_publisher |
| rospy.Subscriber | rclpy Node.create_subscription |
| rosparam | ROS2 parameters |
| MoveIt1 | MoveIt2 |

这样你不会被 ROS1 锁死。

### 研二：把自己的模块写成可迁移结构

这一步很关键。

比如你做的是 LLM/VLA + 机械臂，不要把代码写死在 ROS1 里。

你可以把系统分成：

```text
核心算法层：Python 普通模块，不依赖 ROS
ROS 接口层：负责收发 topic/service/action
机械臂执行层：调用 MoveIt 或 SDK
```

这样将来迁移 ROS2 时，不需要全部重写。

例如：

```text
task_planner.py        # LLM/VLA 任务规划，不依赖 ROS
state_machine.py       # 状态机/监督控制，不依赖 ROS
vision_detector.py     # 视觉检测，尽量少依赖 ROS
ros1_interface.py      # ROS1 接口
ros2_interface.py      # 以后可替换成 ROS2 接口
```

这个思路非常重要。  
它能让你既能在组里跑 ROS1，又不被 ROS1 绑死。

这对你未来很有价值。

## Python 学到哪就能写？

你只要学到这些内容就能写最简单版本：

```text
1. 字符串 str
```

不需要机器学习，不需要深度学习，不需要 ROS。

## 最简单版本长这样

```python
def parse_instruction(text):
    if "红色" in text and "方块" in text and "盒子" in text:
        return [
            {"action": "detect", "object": "red_block"},
            {"action": "pick", "object": "red_block"},
            {"action": "place", "target": "box"}
        ]

    elif "蓝色" in text and "方块" in text and "左边" in text:
        return [
            {"action": "detect", "object": "blue_block"},
            {"action": "pick", "object": "blue_block"},
            {"action": "place", "target": "left_area"}
        ]

    else:
        return [
            {"action": "unknown", "message": "暂时无法解析这条指令"}
        ]


command = input("请输入机械臂指令：")
result = parse_instruction(command)

print(result)
```

运行时：

```text
请输入机械臂指令：把红色方块放到盒子里
```

输出：

```python
[
    {'action': 'detect', 'object': 'red_block'},
    {'action': 'pick', 'object': 'red_block'},
    {'action': 'place', 'target': 'box'}
]
```

## 这和机械臂有什么关系？

机械臂本身听不懂：


## 1. 先把机械臂简化成这样

假设机械臂只有两段：

```text

```

其中：

```text
O：机械臂底座
A：第一个关节末端
P：机械臂末端
L1：第一根杆长度
L2：第二根杆长度
θ1：第一根杆相对水平线的角度
θ2：第二根杆相对第一根杆的角度
```

你要做的事情就是：

> 已知 L1、L2、θ1、θ2，算出 A 点和 P 点坐标，然后画出来。

---

## 2. 核心公式就这几行

第一段末端 A 点：

```text
x1 = L1 * cos(θ1)
y1 = L1 * sin(θ1)
```

第二段末端 P 点：

```text
x2 = x1 + L2 * cos(θ1 + θ2)
y2 = y1 + L2 * sin(θ1 + θ2)
```

为什么是 `θ1 + θ2`？

因为第二根杆的角度不是相对地面的，而是相对第一根杆的。  
所以第二根杆相对地面的总角度是：

```text
θ1 + θ2
```

---

## 3. 最简单的代码

你只需要会：

```text
变量
函数
math.sin / math.cos
matplotlib 画图
```

代码如下：

```python
import math
import matplotlib.pyplot as plt


def forward_kinematics(l1, l2, theta1_deg, theta2_deg):
    """
    二连杆机械臂正运动学：
    已知两段杆长和两个关节角，计算各关节点坐标。
    """

    # 角度转弧度，因为 Python 的 sin/cos 用的是弧度
    theta1 = math.radians(theta1_deg)
    theta2 = math.radians(theta2_deg)

    # 底座坐标
    x0, y0 = 0, 0

    # 第一段末端坐标
    x1 = l1 * math.cos(theta1)
    y1 = l1 * math.sin(theta1)

    # 第二段末端坐标
    x2 = x1 + l2 * math.cos(theta1 + theta2)
    y2 = y1 + l2 * math.sin(theta1 + theta2)

    return (x0, y0), (x1, y1), (x2, y2)


def draw_arm(p0, p1, p2):
    """
    画出二维机械臂
    """

    x_values = [p0[0], p1[0], p2[0]]
    y_values = [p0[1], p1[1], p2[1]]

    plt.figure(figsize=(6, 6))

    # 画机械臂连杆
    plt.plot(x_values, y_values, marker="o", linewidth=3)

    # 标注点
    plt.text(p0[0], p0[1], " Base")
    plt.text(p1[0], p1[1], " Joint")
    plt.text(p2[0], p2[1], " End")

    # 坐标范围
    plt.xlim(-3, 3)
    plt.ylim(-3, 3)

    # 保持横纵比例一致
    plt.gca().set_aspect("equal", adjustable="box")

    plt.grid(True)
    plt.title("2-Link Planar Robot Arm")
    plt.xlabel("X")
    plt.ylabel("Y")

    plt.show()


# 设置两段机械臂长度
L1 = 1.5
L2 = 1.0

# 设置两个关节角度
theta1 = 45
theta2 = 60

# 计算坐标
base, joint, end = forward_kinematics(L1, L2, theta1, theta2)

print("底座坐标:", base)
print("关节坐标:", joint)
print("末端坐标:", end)

# 画图
draw_arm(base, joint, end)
```

```text
变量
列表
字典
函数
```

这几个已经足够写很多基础程序了。  
如果你现在继续猛看后面的类、文件、异常、模块、第三方库，可能会出现一个问题：

> 学得越来越多，但每一块都只会看，不会用。

所以你现在最应该做的不是继续狂看新视频，而是停下来用 3～5 天，把前面的东西真正练熟。

# 二、你应该怎么回顾前面的知识？

不要用“重新看一遍视频”的方式回顾。

那样很低效，因为你看视频时会产生一种错觉：

> 这个我懂了。

但真正判断懂不懂，要看你能不能不看视频写出来。

我建议你用这 4 步回顾法。

---

## 第一步：列知识点清单

你拿一张纸，或者建一个文档，写下：

```text
变量：
- 什么是变量？
- int、float、str、bool 是什么？
- input 得到的是什么类型？
- 怎么做类型转换？

列表：
- 怎么创建列表？
- 怎么取元素？
- append、remove、pop、len 怎么用？
- 怎么遍历列表？

字典：
- key 和 value 是什么？
- 怎么取值？
- 怎么新增、修改、删除？
- 怎么遍历字典？

函数：
- def 怎么写？
- 参数是什么？
- return 是什么？
- print 和 return 有什么区别？
```

这个清单不是为了背，而是为了查漏。

---

## 第二步：每个知识点写 3 行小代码

比如变量，你写：

```python
age = 22
name = "小明"
print(name, age)
```

列表：

```python
fruits = ["apple", "banana", "orange"]
fruits.append("pear")
print(fruits)
```

字典：

```python
student = {"name": "小明", "age": 22}
student["major"] = "control engineering"
print(student)
```

函数：

```python
def add(a, b):
    return a + b

print(add(3, 5))
```

不要嫌简单。

你现在要训练的是：

> 我能不能不看视频，把最基础的语法写出来。

---

## 第三步：遮住原代码，重新写

这个非常重要。

你先看一遍例子，然后把它关掉。  
自己重新写一遍。

如果写不出来，就说明你只是“看懂”，还没“掌握”。

你可以这样循环：

```text
看一遍
↓
手敲一遍
↓
关掉
↓
自己重写
↓
报错就改
↓
再重写一遍
```

这一步会让你很痛苦，但它最有效。

---

## 第四步：把多个知识点组合起来

比如你学了列表、字典、函数，就写一个“学生信息管理”的极简版本：

```python
students = []


def add_student(name, age):
    student = {"name": name, "age": age}
    students.append(student)


def show_students():
    for student in students:
        print(student["name"], student["age"])


add_student("小明", 22)
add_student("小红", 21)

show_students()
```

这不是什么大项目，但它已经把：

```text
列表
字典
函数
for 循环
```

组合起来了。

你以后写程序就是不断做这种组合。

# 三、后边内容应该怎么学？

我建议你接下来不要按“视频顺序”平均学，而是按下面这个顺序学。

## 第一阶段：先补控制流

你已经学了变量、列表、字典、函数。  
接下来必须把这几个补牢：

```text
if / elif / else
for 循环
while 循环
字符串处理
```

这几块是你“会不会写逻辑”的核心。

特别是：

```text
if：负责判断
for：负责重复处理
函数：负责封装功能
列表/字典：负责存数据
```

这几个一组合，你就能写出很多东西。

---

## 第二阶段：学文件和 json

之后学：

```text
文件读写
json
异常处理 try/except
```

为什么这几个重要？

因为以后你做机械臂、ROS、LLM，经常会遇到配置文件、实验记录、动作序列。

比如动作序列可以保存成 JSON：

```json
[
  {"action": "detect", "object": "red_block"},
  {"action": "pick", "object": "red_block"},
  {"action": "place", "target": "box"}
]
```

你以后会经常用 Python 读写这种数据。

所以 `json` 很重要。

---

## 第三阶段：学模块和包

然后学：

```text
import
自己写模块
包的概念
pip
conda
虚拟环境
```

这一步决定你能不能看懂真实项目。

视频里的代码通常是一个文件，但真实项目是很多文件：

```text
main.py
planner.py
vision.py
robot.py
utils.py
config.json
```

你要慢慢适应这种结构。

---

## 第四阶段：类 class 不要太早深挖

类当然重要，但你现在不用急着把面向对象学得特别深。

你先学到能看懂这种就行：

```python
class RobotArm:
    def __init__(self):
        self.position = [0, 0, 0]

    def move_to(self, x, y, z):
        self.position = [x, y, z]
        print("移动到", self.position)
```

你只要先理解：

```text
class 是把数据和功能放在一起
self 表示这个对象自己
__init__ 是初始化
方法就是类里的函数
```

够了。

等你后面看 ROS、机械臂 SDK、OpenCV 项目时，会慢慢熟悉 class。

# 五、你现在最该练的不是大项目，而是“半独立练习”

你现在直接写完整项目，肯定磕磕绊绊。

## 第四步：把中文翻译成 Python

| 中文想法 | Python |

| 存一堆东西 | list |

| 返回结果 | return |
| 显示结果 | print |

你一开始就按这个翻译。

# 七、你怎么判断自己学会了？

不要用“我看懂了”判断。

用下面四个标准：

## 标准一：能不看视频写出例子

比如学了函数，你能不能自己写：

```python
def square(x):
    return x * x
```

能写出来，才算初步会。

---

## 标准二：能改例子

视频里是加法函数：

```python
def add(a, b):
    return a + b
```

你能不能改成：

```python
def multiply(a, b):
    return a * b
```

能改，说明开始理解了。

---

## 标准三：能解释每一行

你能不能用自己的话解释：

```python
for action in actions:
    execute_action(action)
```

如果能解释：

```text
从 actions 这个列表里，一个一个取出动作，每次把动作传给 execute_action 函数执行
```

说明你真的懂。

---

## 标准四：能隔一天重写

今天会不算会。  
明天不看还能写出来，才算更稳。

# 十、给你一个特别重要的建议

你现在不要追求“完整写个东西”。

你先追求：

> **我能不能写出一个函数。**

很多程序都是由函数拼起来的。

比如你以后做一个机械臂任务程序，可能是：

```python
command = input("请输入指令：")

actions = parse_command(command)

for action in actions:
    execute_action(action)
```

这里面最重要的是两个函数：

```python
parse_command(command)
execute_action(action)
```

你先不要管整个系统，先练会一个个小函数。

# 最后总结一下你的学习方法

你现在的学习路线应该是：

```text
不要急着看后面
↓
先用 3～5 天回顾变量、列表、字典、函数
↓
每个知识点都要不看视频重写

↓
再学文件、json、异常
↓
再学模块、虚拟环境
```

每天学习方法：



你现在这个阶段最重要的一句话是：

> **不要从“我要写完整程序”开始，而是从“我能不能写一个小函数”开始。**

只要你能把一个个小函数写出来，后面自然就能组合成完整程序。

## 一、LeetCode 对你有什么帮助？

### 1. 帮你把 Python 基础用熟

你现在学了变量、列表、字典、函数，但不会独立写东西。  
LeetCode 简单题可以逼你用这些语法。

比如：

| Python 知识 | LeetCode 会逼你练什么 |
|---|---|
| 列表 | 遍历、查找、统计、修改 |
| 字典 | 计数、映射、查重 |
| 字符串 | 分割、判断、匹配 |
| if/for | 条件判断、循环逻辑 |
| 函数 | 输入参数、返回结果 |

所以它能帮你从：

> “我看得懂代码”

变成：

> “我能写出一段逻辑”。

---

### 2. 帮你训练“拆题能力”

LeetCode 每道题其实都是在训练你：

```text
输入是什么？
输出是什么？
中间怎么处理？
有哪些特殊情况？
```

这和你以后写机械臂程序也有关。

比如以后机械臂任务是：

> 判断动作序列是否合法。

本质上也是：

```text
输入：动作列表
输出：合法 / 不合法
处理：检查动作顺序、状态变化、异常情况
```

这和算法题里的状态判断、列表遍历、条件分支是相通的。

---

### 3. 帮你提高写代码的熟练度

刚开始你写代码磕磕绊绊，很正常。

LeetCode 简单题刷到一定量后，你会发现：

```text
for 循环不再陌生
if 判断写得更顺
字典计数知道怎么写
列表遍历知道怎么处理
函数返回值更清楚
```

这对你后面写 ROS 节点、视觉处理脚本、机械臂状态机都有帮助。

## 每道题按 4 步来

### 第一步：先不用代码，写中文思路

比如题目是“两数之和”。

你先写：

```text
输入：一个数字列表 nums，一个目标值 target
输出：两个下标
方法：遍历每个数字，看 target - 当前数字 是否出现过
用字典保存数字和下标
```

这一步比直接写代码重要。

---

### 第二步：自己写第一版，哪怕很笨

哪怕你写暴力解法也可以：

```python
for i in range(len(nums)):
    for j in range(i + 1, len(nums)):
        if nums[i] + nums[j] == target:
            return [i, j]
```

先写出来，再优化。

---

### 第三步：看题解，但不要复制

你可以看题解，但看完要关掉，然后自己重写。

如果你只是复制题解，那提升很小。

---

### 第四步：隔天重写

这一步最重要。

今天做会了，不代表会了。  
明天不看答案还能写出来，才算真的掌握。

# 四、适合你现在刷的题型

我建议你先刷这些类型。

## 1. 数组遍历类

训练 for 循环和列表。

比如：

```text
两数之和
移除元素
合并两个有序数组
买卖股票的最佳时机
最大子数组和，先了解即可
```

重点不是算法多高级，而是熟悉列表操作。

---

## 2. 字符串类

训练字符串处理。

比如：

```text
有效的字母异位词
反转字符串
最长公共前缀
验证回文串
```

这些对你以后处理文本指令也有点帮助。

---

## 3. 哈希表类

训练字典。

比如：

```text
两数之和
有效的字母异位词
多数元素
存在重复元素
两个数组的交集
```

字典非常重要。  
以后机器人系统里你会经常用字典描述状态、动作、物体。

---

## 4. 栈和队列入门

训练状态变化。

比如：

```text
有效的括号
用队列实现栈，了解即可
最小栈，后面再看
```

“有效的括号”这种题，其实和动作序列合法性有点像。

比如机械臂：

```text
pick 之后才能 place
open_gripper 和 close_gripper 要成对
```

这类状态判断以后会用到。

# 七、怎么判断 LeetCode 对你产生明显提升了？

你会出现这些变化：

```text
看到列表题，知道先 for 遍历
看到统计题，想到用字典
看到匹配题，想到用栈
看到查找题，想到用 set 或 dict
写函数时知道 return 什么
报错时能自己定位
```

还有一个很明显的标志：

> 你能不看视频，自己写出 30～50 行的小脚本。

这时候 LeetCode 的作用就体现出来了。

# 八、给你一个非常实际的刷题目标

你现在不要定：

> 我要刷完 300 道。

这个太虚，也没必要。

你先定：

> **用 30 道简单题，把 Python 基础写顺。**

这 30 道最好覆盖：

```text
数组 10 道
字符串 8 道
哈希表 8 道
栈 4 道
```

刷完 30 道，你再决定要不要继续。

# 九、最终建议

对你来说，LeetCode 有用，但它的位置应该是：

> **Python 熟练度训练工具，不是研究生预习主线。**

你现在最应该的顺序是：

```text
先把 Python 基础语法补齐
↓
刷 20～30 道简单 LeetCode，练语法应用
↓
开始做小脚本和小项目
↓
转向 Linux、ROS、机器人学
↓
LeetCode 低频保持
```

一句话：

> **当你学完 if、for、while、列表、字典、函数后，刷简单 LeetCode 对你的提升最明显。**

但不要沉迷刷题。  
你最终要服务的是机械臂、ROS、控制工程和研究项目，不是成为纯算法刷题选手。
